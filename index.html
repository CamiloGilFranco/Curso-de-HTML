Etiquetas Meta

Las etiquetas meta son invisibles para el usuario final pero son de vital importancia para navegadores y motores de busqueda ya que preveen la informacio general de la pagina

METAETIQUETAS BASICAS
<meta charset="UTF-8"> Hace que la pagina soporte todos los caracteres
<meta http-equiv="X-UA-compatible" content="IE=edge"> Esta etiqueta no es necesaria, se usa para darle compatibilidad a internet explorer
<meta name="viewport" content="width=device-width, initial-scale=1.0"> Ajusta el ancho del contenido al ancho de la pagina
<title> Es el titulo de la pagina, este titulo debe tener maximo 65 carateres
<meta name="description" content="contenido"> Es donde se escribe la descripcion de la pagina

METAETIQUETAS PARA SEO
Son etiquetas para posicionamiento en buscadores
<link rel="canonica" href="URL canonica o principal"> Permite informar a los buscadores cual es la pagina principal (no afectada por variables)
<link rel="icon" href="ruta hacia una imagen"> Permite agregar una imagen como icono de la pagina, esta imagen debe ser cuadrada y en formato PNG
<link rel="apple-touch-icon" href="ruta hacia una imagen"> Permite dar un icono a la pagina en caso de ser guardada en un dispositivo movil, esta imagen tambien debe ser cuadrada y en formato PNG
<meta name="theme-color" content="color en valor hexadecimal"> Permite cambiar el color de la barra de direcciones en dispositivos moviles

METAETIQUETAS PARA REDES SOCIALES
-Dan la informacion a las redes sociales para crear tarjetas de enlace cuando la URL de mi sitio sea compartida en ellas
-Cada red social tiene diferentes opciones para crear estas tarjetas, estas opciones se pueden buscar en la seccion de desarrollador de cada red social
-La mayoria de las redes sociales usa el protocolo opengraf que son las que se explican a continuacion
<meta property="og:title" content="titulo visible por el usuario"> Pone el titulo a la tarjeta
<meta property="og:description" content="descripcion visible por el usuario"> Pone descripcion a la tarjeta
<meta property="og:image" content="URL donde esta la imagen en internet"> Pone imagen a la tarjeta, no se debe usar la ruta a la carpeta local sino a la url donde esta la imagen, por lo tanto, la pagina ya debe estra en un servidor en linea
<meta property="og:url" content="URL a la pagina"> Garantiza que tarjeta lleve a la URL correcta

estilos CSS en HTML

<style>
La etiqueta <style> sirve para dar estilo como color a secciones determinadas, sin embrgo, si esta etiqueta se escribe en la seccion <head> del documento, da estilo a grupos de etiquetas escribiendolo de la siguiente forma: 

    <style>
    h2 {
    color: orangered;
    }
    </style>

La forma correcta de aplicar estilos es mediante CSS. Para esto se crea una nueva carpeta dentro del proyecto llamada assets y en ella un archivo estilos.css alli se aplican todos los estilos al documento html.

Para enlazar la oja de estilos, en la seccion <head> usar la etiqueta <link rel="stylesheet" href="assets/estilo.css">
el atributo rel hace referencia al tipo de documento, datasheet representa hoja de estilos.
el atributo href hace referencia a la ruta donde se encuentra el documento a importar

scripts en HTML

<script>

La etiqueta </script> se usa para ejecutar programacion hecha en JavaScript. Esta programacion se hace en un documento aparte.

Para crear este archivo, primero se debe crear una nueva cartpeta con el nombre assets y dentro de esta crear un archivo con el nombre codigos.js

para importar un documento JS, se usa la etiqueta <script src="assets/codigos.js"> donde en el atributo src se escribe la ruta donde esta el documento JS, y dentro de la etiqueta de apertura y cierra no se debe poner nada.

es importante poner las etiquetas </script> al final del documento html.

imagenes

antes de insertar imagenes en el documento se debe crear una carpeta de recursos
Los formatos admitidos para imagenes son: JPG, PNG, GIF, SVG(vectores)

<img src="" alt="">
En el atributo src se escibe la ruta de la imagen a utilizar
en el atributo alt se escribe el titulo de la imagen, este titulo se mostrara en caso de que la imagen no sea cargada.
para redimencionar la imagen se pueden usar los atributos width="" o height="" sin embargo la forma correcta de hacerlo es mediante la hoja de estilos CSS como se muestra a continuacion. 

    img,
    video {
    max-width: 100%;
    height: auto;
    }
esta estructura hace que el tamaño maximo de la imagen tenga la anchura de la ventana y el tamaño minimo sea el que tiene la imagen por defecto en caso de este ser mas angosto que la ventana.
como se puede ver en el ejemplo esta misma estructura se usa para redimencionar videos.

enlaces

<a href=""> Genera enlaces hacia otros documentos html. Entre la etiqueta de apertura y cierre se debe escribir el texto que vera el usuario

Para documentos dentro del proyecto se debe escribir la ruta hacia este en el parametro href. Ejemplo: <a href="hola.html">
click para ir a la pagina hola

Para paginas externas se debe escribir en el parametro href la pagina de destino con el protocolo http//: o https//: segun corresponda. Ejemplo: <a href="https://jonmircha.com">
Seguir enlace en la misma pestaña

se puede usar el parametro target="_blank" para que el enlace se abra en una pestaña nueva. ejemplo: <a href="https://jonmircha.com" target="_blank">
Seguir el enlace en una nueva pestaña

Se puede usar una imagen como boton para seguir al enlace. Para esto se debe insertar una imagen dentro de las etiquetas <a> </a>. Ejemplo:
<a href="https://jonmircha.com" target="_blank"><img src="img/1.jpg" alt="nadita" width="40" height="40"><a>

Audio y video

En estas etiquetas no se debe poner nada entre la etiqueta de apertura y cierre, todas las modificaciones se hacen con atributos.

<audio>
Permite poner piezas de audio en el sitio.
Se debe agregar el atributo controls para que el reproductor se pueda manipular.
Al agregar el atributo autoplay la pista se iniciara al cargar la pagina pero esto solo funciona en navegadores de escritorio.
Al usar el atributo loop una vez se termine la pista, esta iniciara de nuevo.
Al usar el atributo muted el reporductor inicia muteado o con volumen en 0.
Al usar el atributo preload se precarga el track en el cache del navegador para tenerlo listo

<video>
Permite poner piezas de video en el sitio, todos los atributos utilizados para la etiqueta <audio> funcionan para esta. Estos son: controls, autoplay, loop, muted, preload
Al usar el atributo poster se puede poner una imagen como portada del video.
Para ajustar el tamaño del video y del poster se pueden usar los mismos atributos usado en la imagenes width y height aunque la forma correcta de hacerlo es mediante la hoja de estilos CSS.

Iframes

Sirve para traer informacion especifica de otro sitio como PDF's, mapas, videos o incluso sitios completos.

<iframe>
El atributo src define la ruta o direccion de donde se importa la informacion.
El atrubuto frameborder define el tamaño de la margen del recuadro donde se aloja la informacion.
Para cambiar el tamaño del recuadro se usa los mismos atributos de las imagenes width y height sin embargo la forma correcta de hacerlo es mediante la hoja de estilos CSS. 

iframe {
width: 80%;
height: 400px;
}


ejemplos:

Documento del mismo proyecto.
<iframe src="hola.html" frameborder="1" width="500" height="300"></iframe>


Documento de un sitio externo
<iframe src="https://jonmircha.com" frameborder="1" width="500" height="300"></iframe>


PDF
<iframe src="media/HV Camilo Gil.pdf" frameborder="1" width="500" height="300"></iframe>

formularios y sus elementos

elementos de formularios

<form>
-dentro de estas etiquetas se escribe todo el Formulario
-el atributo action define la direccion a donde seenviara el formulario
-el atributo method define el protocolo de envio del formulario. existen dos protocolos. get: envia los valores por la URL. post: envia las variables de forma oculta.
-una vez enviado el formulario, en la direccion de destino se mostrara el simbolo ? a partir de ahi se muestran las variables que fueron enviadas
-si el atributo action no se scribe, los valor del formulario se enviaran al mismo documento de donde salieron

-el atributo for sirve para vincular el label con el campo, se debe tener en cuenta que el campo debe tener el atributo id
-los radiobutton y los checkbox deben tener un label cada uno o el formulario perdera sentido semantico
-se puede vincular el campo con su label poniendo la etiqueta <input> dentro de la etiqueta <label>. Ejemplo: <label><input type="checkbox" namme="interese" value="deportes">Deportes</label>.

-para que esto funcione la pagina debe estar en un servidor real, sin embargo, se puede intalar en Visual Studio Code la extencion live server para emular un servidor. -una vez intalada la extencion, se da click derecho en cualquier parte del codigo y luego click en open with live server
